How P19 Works
When a user approves an action, VisIQ generates an authorization artifact (R1) — a cryptographically signed record of exactly what was approved. At execution, VisIQ intercepts the agent's outbound action (R2) and validates it against R1. If R2 does not match R1 in scope, parameters, resource, or identity — execution is blocked before it reaches any external system. Not monitored. Not logged. Blocked.

P19 also detects Cl. 10 — Deceptive Framing: when an agent describes its intended action misleadingly to obtain approval, then executes a materially different action at runtime. Confidence scores below 0.25 on the alignment vector trigger automatic blocking regardless of whether the action falls within nominal scope boundaries.

Fail-Safe Behavior
If any dimension has changed materially since approval, execution is blocked automatically — without requiring human intervention. The agent must re-request authorization under the new conditions.

If the CVD validation subsystem itself is unavailable for any reason: execution is blocked. Fail closed. Always.

SDK — Modules
Each module is independently embeddable. Use one or all four.
Policy Engine
Runtime rule evaluation · <5ms · fail-closed
ALLOW
Define rules in YAML or the API. Every agent action is intercepted and validated at request time — permit, deny, or escalate for manual review — before any external system is touched. Rules compose with AND/OR logic, support attribute-based conditions, and chain to downstream actions like incident logging or agent suspension.
PythonTypeScriptJavaGoREST API
from visiq import PolicyEngine pe = PolicyEngine(org_id="org_demo") decision = pe.evaluate( agent="my-agent", action="wire_transfer", context={"amount": 15000, "currency": "USD"} ) # decision.result → "deny" (rule: amount > $10K)
Escalation Gateway
Intercept · pause · route · resume
Escalate
When a rule triggers escalation, the agent is suspended mid-execution. The action, full context, identity lineage, and request payload are routed to your approval queue. On approval, the agent resumes with a signed authorization artifact. On rejection, the agent is blocked. Integrates with Slack, email, PagerDuty, or custom webhooks.
WebhookSlackPagerDutyEmailREST API
const gw = new EscalationGateway({ org: 'org_demo', notify: { slack: '#agent-approvals' } }); // In your agent executor: const gate = await gw.intercept(agentAction); if (gate.status === 'pending') { // Agent automatically suspended await gate.waitForDecision(); }
Runtime Trust
P19 Intent Validation · P20 CVD · at t-exec
TRUST
The only module on the market that re-validates authorization at the moment of execution. P19 confirms the agent's outbound action matches what was approved (intent alignment). P20 re-scans the contextual fingerprint across four dimensions — if the world changed since approval, execution is blocked automatically. Fail-closed by design.
PythonTypeScriptJavaGoRust SDK
from visiq.trust import RuntimeTrust rt = RuntimeTrust(artifact=approval_artifact) # Call at exact execution moment result = rt.validate_at_exec( actual_action=agent.pending_call, context_snapshot=env.snapshot() ) # result.p19 → PASS | BLOCK (scope drift) # result.p20 → VALID | CVD-BLOCK (state changed)
Audit Chain
Ed25519 signed · tamper-evident · exportable
RECORD
Every decision — release, block, escalate, CVD-block — generates a cryptographically signed receipt. Receipts are chained (each references the previous), making tampering or gap insertion mathematically detectable. Export monthly compliance packages in JSON with Ed25519 signatures for SOC 2, HIPAA, and financial audit requirements.
JSONEd25519SOC 2HIPAAREST API
const chain = new AuditChain({ org: 'org_demo' }); // Auto-called by other modules — or call directly: const receipt = await chain.record({ decision: 'deny', agent: 'finance-bot', action: 'wire_transfer', rule: 'Block >$10K' }); // receipt.sig → Ed25519:a3f9c12b…
